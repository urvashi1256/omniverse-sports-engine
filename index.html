Omniverse Sports Engine

Underdog-goal signal engine for football prediction markets

Engine overview

The engine watches for goal events in football matches, flags cases where the scoring team is an underdog, and then fires a BUY signal to capture the price move that usually follows a goal shock.

Live score feed → Match store → Decision engine → Order executor ↓ ↓ (underdog?) Trades / logs

Repository & markets targeted

Code & docs: github.com/urvashi1256/omniverse-sports-engine

  • Engine name: Omniverse Football Underdog Engine
  • Markets targeted: football / soccer match-winner markets on prediction exchanges (e.g. Kalshi / Polymarket–style YES/NO contracts).

How to run the demo

  1. Clone the repo: git clone https://github.com/urvashi1256/omniverse-sports-engine.git
  2. Enter the folder: cd omniverse-sports-engine
  3. Install dependencies: pip install -r requirements.txt
  4. Run a single simulated goal signal: python demo.py
  5. Run the looping engine: python main.py (Ctrl+C to stop)

Architecture & components

  • Score feed / normalizer: simplified in this version as an in-memory goal writer.
  • Decision engine: tags a scoring team as underdog based on configurable rules.
  • Order executor: prints simulated BUY orders and would plug into an exchange API.

Known limitations

  • Uses a synthetic goal feed instead of live SofaScore / Livescore scraping.
  • Order placement is simulated; no real Kalshi / Polymarket orders are sent.
  • No persistence layer yet; state is kept in memory for clarity.
